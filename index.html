¿Qué es un Lenguaje?

Un diálogo estructurado entre el ser humano y la máquina. En el contexto móvil multiplataforma, su función crítica es abstraer la complejidad del hardware manteniendo la expresividad del código.

Arquitectura Lingüística

  • 📜
    Sintaxis: La gramática y reglas de escritura.
  • 🧠
    Semántica: El significado lógico detrás del código.
  • ⚙️
    Pragmática: La implementación práctica y eficiencia.

Niveles de Abstracción

Bajo Nivel Ensamblador / Máquina

Sin abstracción. Orientado al procesador. Difícil mantenimiento.

⬇️
Alto Nivel (2025) ESTÁNDAR

Diseñados para humanos. Facilitan lógica compleja.

Ej: Kotlin, Dart, TypeScript

🏛️ Orientación a Objetos (POO)

Une datos y procesos en objetos. Fundamental para gestionar la complejidad de interfaces modernas mediante:

  • ✅ Herencia: Reutilización de código.
  • ✅ Polimorfismo: Flexibilidad de tipos.
  • ✅ Encapsulamiento: Seguridad de datos.

⚡ Programación Funcional

Clave en Swift y Kotlin para concurrencia. Introduce conceptos vitales para flujos asíncronos sin bloqueos:

  • ✅ Inmutabilidad: Datos que no cambian.
  • ✅ Lambdas: Funciones anónimas flexibles.
  • ✅ Flujos de datos: Manejo reactivo eficiente.

💡 Caso Dart (Flutter)

Optimizado para UI. Su fortaleza es el sistema de tipos y una recolección de basura (GC) eficiente en memoria limitada.

💡 Caso Kotlin (KMP)

Destaca por su Null Safety (seguridad ante nulos) e interoperabilidad total con Java. Estándar para lógica compartida.

Compilación y Ejecución

El proceso de transformar código fuente en instrucciones ejecutables define la velocidad y productividad del desarrollo.

📄

Interpretado

Traducción línea a línea durante la ejecución.

✅ Flexibilidad total
❌ Rendimiento lento
⚡

JIT

Compilación durante la ejecución (bytecode → máquina).

✅ Hot Reload, iteración rápida
❌ Sobrecarga de memoria
🚀

AOT

Compilación antes de la ejecución (binario nativo).

✅ Máximo rendimiento, inicio rápido
❌ Tiempos de compilación largos

Comparativa de Modelos

Modelo Momento de Traducción Ventaja Principal Desventaja Principal
Interpretado Durante ejecución
(línea a línea)		 Flexibilidad total Rendimiento lento
JIT Durante ejecución
(bloques de código)		 Hot Reload, iteración rápida Sobrecarga de memoria
AOT Antes de ejecución
(fase de construcción)		 Máximo rendimiento, inicio rápido Tiempos de compilación largos

Caso Flutter: Lo Mejor de Ambos Mundos

Utiliza JIT durante el desarrollo para favorecer la agilidad (Hot Reload) y AOT para producción para garantizar el máximo rendimiento en la app final.

🤖 Entorno Android

📦 JDK & SDK

Instalar Oracle JDK o OpenJDK compatible y configurar la variable de entorno crítica: 

export JAVA_HOME=/ruta/al/jdk

⚙️ Android SDK

  • 📂 ANDROID_HOME: Apuntar al directorio raíz del SDK
  • 🛣️ PATH: Incluir tools, tools/bin y platform-tools
  • 🔧 Android Studio: Gestor recomendado para descarga del SDK

🍏 Ecosistema Apple

🛠️ Xcode & CLI

Primer paso: instalar Xcode desde App Store. Luego, ejecutar en terminal: 

xcode-select --install

🐚 CocoaPods (Ruby)

Gestor de dependencias para iOS. ⚠️ Apple Silicon (M1/M2/M3):

  • ⚠️ Conflictos posibles con Ruby del sistema
  • ✅ Usar RVM o rbenv para entorno aislado
  • 🔒 Garantiza estabilidad en la gestión de librerías

💡 Ejemplo: Consistencia en Nomenclatura

// ❌ Evitar:
var userName:String="John"
fun GetUser(){...}

// ✅ Preferir (Kotlin Style):
val userName: String = "John"
fun getUser() { ... }
// ❌ Evitar (JS):
var count = 0;
function GetData() { ... }

// ✅ Preferir (Airbnb Style):
const count = 0;
const getData = () => { ... };
Aplicar estas reglas automáticamente con hooks de pre-commit o integración en CI/CD.

Arquitectura Limpia y Estrategias de Repositorio

Más allá del estilo: los patrones arquitectónicos y la organización del código definen la escalabilidad a largo plazo.

🏗️ Clean Architecture + MVVM

🎯 Separación de Responsabilidades

La lógica de negocio debe ser independiente de la UI y de las fuentes de datos externas.

🔄 Flujo Unidireccional

El estado fluye en una sola dirección, facilitando la depuración y trazabilidad de errores.

📊 SSOT Single Source of Truth: Repositorios centralizados para gestión de datos
➡️ UDF Unidirectional Data Flow: Estado predecible y trazable
💉 DI Dependency Injection: Hilt/Dagger para desacoplar componentes
🗂️

Monorepo

Un único repositorio para múltiples proyectos y librerías compartidas.

  • ✅ Coherencia en dependencias
  • ✅ Refactorización a gran escala simplificada
  • ✅ Visibilidad global del código
  • ⚠️ Requiere herramientas avanzadas: Nx, Bazel
📦

Polyrepo

Múltiples repositorios independientes para cada proyecto o módulo.

  • ✅ Autonomía total de los equipos
  • ✅ Ciclos de lanzamiento independientes
  • ✅ Menor complejidad inicial
  • ⚠️ Mayor dificultad en coordinación de versiones
